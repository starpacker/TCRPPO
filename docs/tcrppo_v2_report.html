1. 框架概述

TCRPPO v2 使用 PPO（Proximal Policy Optimization） 强化学习算法， 通过序列编辑操作（替换/插入/删除）来设计 T细胞受体 CDR3β 序列。 目标是同时满足三重约束：高靶标亲和力、低脱靶交叉反应和高自然度。

🔑 核心设计要素

🎯
tFold (FP32) 亲和力评分
基于结构的结合亲和力评分器，输出 binding logit（非 sigmoid）。FP32 精度确保评分一致性。735M 参数特征提取器 + 1.57M 分类器。
🔄
Online Dynamic Pool
训练中持续收集高质量 TCR 到在线池，dynamic band selection 基于近期表现自动调整采样范围。
🏁
Terminal Reward
每个 episode 结束（8步后）才给予一次性奖励。terminal reward 信号更干净—
🧬
AE+GMM 自然度评分
Autoencoder 重建精度 + GMM 潜空间似然度组合评分，检测 poly-C 等不自然序列。阈值 0.8 以上无惩罚，低于产生负惩罚。
🛡️
Decoy 特异性约束
Tier A+B decoy 肽段（1-3 AA 突变体），K=2 采样，确保 TCR 不对相似肽段交叉反应。
🧊
ESM-2 (650M) 冻结编码器
预训练蛋白质语言模型提取 1280 维特征，frozen 模式不需反向传播。

📐 奖励函数

Reward Mode: v2_simple_target_gated_decoy

R = w_aff × A_target − w_decoy × DecoyPenalty + w_nat × P_nat + w_div × P_div + bonus × 𝟙[A_target > gate]
  • A_target: tFold FP32 输出的 binding logit（可正可负，正值表示结合倾向）
  • DecoyPenalty: 当 A_target ≥ gate 时才计算，K=2 个 Tier A+B decoy 的平均亲和力
  • P_nat: AE+GMM 组合分 ≥ 0.8 → 0（无惩罚），< 0.8 → 负值惩罚
  • P_div: 序列多样性奖励（Hamming 距离）
  • bonus: A_target 超过 gate 时的固定奖励

2.2 标准 MLP 策略网络（ActorCritic）

用于 trace94, trace98, trace99。三头自回归动作空间：

obs [B, obs_dim] │ ▼ Backbone: Linear(obs_dim→512) → ReLU → Linear(512→512) → ReLU │ ├──▶ Head 1: op_type (4-way) [SUB / INS / DEL / STOP] │ Linear(512→4) + action masking → sample → op │ ├──▶ Head 2: position (max_len=20) conditioned on op │ op → Embedding(4,32) → op_emb │ [features|op_emb] → Linear(544→256) → ReLU → Linear(256→20) │ + pos_mask → sample → pos │ ├──▶ Head 3: token (20 amino acids) conditioned on op + pos │ pos → Embedding(20,32) → pos_emb │ [features|op_emb|pos_emb] → Linear(576→256) → ReLU → Linear(256→20) │ + token_mask (仅 SUB) → sample → tok │ └──▶ Value Head (Critic) Linear(512→256) → ReLU → Linear(256→1)

自回归特性：position 以 op 为条件，token 以 op+position 为条件。 orthogonal init (gain=√2)，策略头 gain=0.01 鼓励初始探索。

2.3 Cross-Attention 策略网络 — trace100

TCR-peptide 交叉注意力机制（4 heads, hidden_dim=256）：

obs → split → TCR_emb[1280] | pMHC_emb[1280] | scalars │ │ ▼ ▼ TCR Encoder Peptide Encoder Linear(1280→256)→ReLU Linear(1280→256)→ReLU │ │ │ Cross-Attention (4 heads, dim=256) │ Q = TCR_features K,V = pep_features │ Multi-head scaled dot-product → attended_pep │ │ ▼ ▼ Fusion: [TCR_feat(256) | attended_pep(256) | pep_feat(256) | scalars] │ (total: 768 + scalar_dim) ▼ Backbone → 3-Head Actor + Value Head（同 §2.2）

设计意图：cross-attention 让 TCR 特征显式"关注"肽段特征， 使策略针对不同靶标产生不同编辑策略。标准 MLP 仅简单拼接 embedding。

6. 合格 TCR 设计结果

筛选标准：target_affinity ≥ 0.6，naturalness ≥ −0.15。 共 29 条合格序列，覆盖 12 个靶标肽段。

6.1 气泡图：Target Affinity × Decoy Affinity × Absolute Naturalness

气泡大小 = absolute naturalness (AE+GMM 原始组合分，未截断)。颜色区分 trace。

6.2 Absolute Naturalness 说明

训练中的 naturalness 使用阈值截断：combined = edit_accuracy + gmm_likelihood， 若 ≥ 0.8 则 penalty=0.0，否则 penalty = combined − 0.8。 Absolute naturalness 用 score_raw() 返回原始组合分（~0-1.5），不截断。

29 条合格 TCR 绝对自然度范围 0.06 ~ 1.31，均值 0.61。 trace98 的短序列（如 CASSVLLHF, 9aa）自然度最高（~1.25）， trace94 的长序列（18aa，含 H/P 重复）自然度低（~0.13-0.29）。

9. 核心发现与结论

🎯 主要结论

  1. tFold FP32 评分空间极稀疏：6 万随机序列零正分，RL ~260K 步可找到多条高分序列（max=1.68）
  2. Naturalness-Affinity trade-off：trace94（低 w_nat）→ 高亲和力+低自然度；trace98（高 w_nat）→ 高自然度+中等亲和力
  3. 从高亲和力 fine-tune naturalness 效果差：trace99 继承高亲和力但 nat_pass=0.7%，策略已陷入不自然局部最优
  4. Naturalness 应从训练初期引入：trace98 从零同时约束，nat_pass 71.2%
  5. Cross-attention 显示早期优势：trace100 在 ~30K 步时优于同期 MLP
  6. Biochem features 提供互补信号：8D 电荷+疏水性特征帮助策略更快学习基本物化模式

🔮 后续方向

  • 继续训练 trace100 到 200K+ 步验证 cross-attention 效果
  • 两阶段训练：先高 w_nat 学自然度，再逐步提升 w_aff
  • Per-residue cross-attention（而非 pooled embedding 级别）
  • 对 29 条合格 TCR 进行 tFold 结构预测验证结合构象
