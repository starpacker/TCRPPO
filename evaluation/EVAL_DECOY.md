# Decoy specificity evaluation

End-to-end pipeline for testing whether TCRs generated by a trained TCRPPO
model are **specific** to their target peptide-MHC, or whether they also
score highly against off-target ("decoy") pMHCs from
[`decoy_library`](https://github.com/starpacker/decoy_library).

The pipeline is split into three scripts so the slow GPU-bound step
(`eval_decoy.py`) only has to run once per ERGO model, and the fast CPU-only
analysis steps can be re-run interactively from a single CSV.

```
eval_decoy.py            ─►  results/decoy/eval_decoy_<ergo>.csv   (heavy, GPU)
   │
   ├──► eval_decoy_metrics.py     ─►  console + per_target_metrics.csv  (fast)
   └──► eval_decoy_visualize.py   ─►  figures/decoy/*.png               (fast)
```

## Layout assumption

`eval_decoy.py` defaults to `../decoy_library` for the decoy repo. The
expected directory layout is:

```
<parent>/
├── TCRPPO/                ← this repo
│   └── evaluation/
│       ├── eval_decoy.py
│       ├── eval_decoy_metrics.py
│       ├── eval_decoy_visualize.py
│       └── ergo_uncertainty.py
└── decoy_library/         ← sibling repo
    ├── data/
    │   ├── candidate_targets.json
    │   ├── decoy_a/
    │   ├── decoy_b/
    │   ├── decoy_c/decoy_library.json   (1900 entries, always loaded)
    │   └── decoy_d/<TARGET>/...
    └── run_decoy.py
```

If you want a different layout, pass `--decoy_library_root /path/to/decoy_library`.

## Step 1 — Generate decoys (one-time, in `decoy_library/`)

Decoy tier C ships pre-built (1900 entries). Tiers A, B and D are generated
on-demand from `decoy_library/run_decoy.py`. To get the full battery of
decoys for the 12 candidate targets:

```bash
cd ../decoy_library
python run_decoy.py --all-targets --tiers a,b,d
```

Tier C is loaded unconditionally; A/B/D are loaded only if their files
exist. You can also disable any tier from the TCRPPO side with
`--no_decoy_a`, `--no_decoy_b`, `--no_decoy_c`, `--no_decoy_d`.

## Step 2 — Run inference + ERGO scoring (GPU)

```bash
cd TCRPPO
python evaluation/eval_decoy.py \
    --ergo_model ae_mcpas \
    --num_tcrs_per_target 50 \
    --num_envs 8 \
    --n_mc_samples 20
```

Without `--model_path`, the script auto-detects the trained PPO checkpoint
under `output/` matching the chosen `--ergo_model` (it will skip
`sanity_check`/`test_run` directories).

To run the same eval against the VDJdb-trained model:

```bash
python evaluation/eval_decoy.py --ergo_model ae_vdjdb --out_csv evaluation/results/decoy/eval_decoy_ae_vdjdb.csv
```

### Useful flags

| Flag | Default | Notes |
|---|---|---|
| `--target_peptides_file` | (use `decoy_library/data/candidate_targets.json`) | Override target list (JSON or one-per-line text) |
| `--max_targets` | `0` (all) | Truncate target list (useful for smoke tests) |
| `--num_tcrs_per_target` | `50` | TCRs sampled from `data/tcrdb/test_uniq_tcr_seqs.txt` per target |
| `--num_envs` | `4` | Vec-env parallelism for PPO rollouts |
| `--n_mc_samples` | `20` | MC dropout forward passes per (TCR, peptide) pair |
| `--ergo_chunk_size` | `4096` | Reduce if you OOM during MC scoring |
| `--dry_run` | off | Load targets + decoys, print stats, skip GPU work |
| `--no_decoy_{a,b,c,d}` | all on | Disable a tier |

### What `eval_decoy.py` does

1. Loads target peptides from `decoy_library/data/candidate_targets.json`
   (or `--target_peptides_file`).
2. Loads decoy peptides from tiers A/B/C/D, deduping by `(peptide, hla)`.
3. Loads the trained PPO model and generates `num_tcrs_per_target` optimized
   TCRs per target via `eval_model.run_inference`.
4. For each target, scores its generated TCRs against the target peptide AND
   the entire decoy pool, using **MC Dropout** on the ERGO model
   (`ergo_uncertainty.mc_dropout_predict_chunked`).
5. Writes one CSV row per `(target, scored_peptide, TCR)` triple.

## Step 3 — Compute metrics

```bash
python evaluation/eval_decoy_metrics.py --csv evaluation/results/decoy/eval_decoy_ae_mcpas.csv
```

Console output sections:

- **Per-target summary** — target ERGO mean, decoy ERGO mean,
  specificity diff, specificity ratio, AUROC (target vs decoy),
  fraction of decoys above the high-binding threshold (default 0.9).
- **Per-peptide top hits** — `top_per_target` highest-scoring decoy
  peptides for each target, with HLA labels (fulfills the per-peptide
  HLA-tagged reporting requirement).
- **Evidence-level breakdown** — decoy ERGO score by IEDB evidence level
  (1/2/3/4 for tier C; "literature"/"sequence_homology"/"structural"/"ai_generated"
  for the other tiers).
- **Uncertainty diagnostics** — target vs decoy MC dropout std,
  Pearson correlation between mean and std (sanity check that the model
  is not equally uncertain on all inputs).
- **Top-K worst off-target hits** — the K decoys with the highest
  ERGO mean across all targets.

CSVs `per_target_metrics.csv` and `per_peptide_metrics.csv` are written
next to the input CSV.

Tunable flags: `--high_threshold` (decoy positivity cutoff, default 0.9),
`--top_k` (top off-target hits, default 15), `--top_per_target`
(per-target hit list size, default 8).

## Step 4 — Plots

```bash
python evaluation/eval_decoy_visualize.py --csv evaluation/results/decoy/eval_decoy_ae_mcpas.csv
```

Output → `evaluation/figures/decoy/`:

- `boxplot_target_vs_decoy.png` — per-target box plot of target ERGO score
  vs the decoy ERGO score distribution.
- `violin_by_hla.png` — per-target violin plots colored by the decoy
  peptide's HLA allele (helps see if off-target hits cluster on one HLA).
- `uncertainty_calibration.png` — scatter of ERGO mean vs MC dropout std,
  colored by target/decoy.
- `top_off_target_hits.png` — bar chart of the K worst decoy hits across
  all targets, annotated with HLA + tier.

Only `matplotlib` + `numpy` are required (no seaborn dependency).

## CSV schema (`eval_decoy_<ergo>.csv`)

| Column | Description |
|---|---|
| `target_pep` | Target peptide sequence |
| `target_hla` | HLA allele the target was generated for |
| `target_source_protein` | Source protein (e.g. `HCV-NS3`, `MAGE-A3`) |
| `decoy_pep` | Scored peptide (= target_pep when `is_target_pep=1`) |
| `decoy_hla` | HLA allele of the decoy peptide |
| `decoy_source_protein` | Source protein of the decoy |
| `decoy_source_tier` | `target` / `A` / `B` / `C` / `D` |
| `decoy_evidence_level` | IEDB evidence level (tier C) or tier label |
| `decoy_origin_target` | Which target the decoy was generated against (tier D only) |
| `init_tcr` | Starting TCR before PPO optimization |
| `final_tcr` | TCR after PPO optimization |
| `edit_dist` | Levenshtein distance from `init_tcr` to `final_tcr` |
| `gmm_likelihood` | TCR-realism log-likelihood from the GMM filter |
| `ergo_mean` | Mean ERGO score over `n_mc_samples` MC dropout passes |
| `ergo_std` | Std-dev of ERGO score across MC passes (uncertainty) |
| `is_target_pep` | `1` if `decoy_pep == target_pep`, else `0` |

## How MC Dropout works here

ERGO's `AutoencoderLSTMClassifier` (`code/ERGO/ERGO_models.py`) has three
`nn.Dropout(p=0.1)` modules in the forward path: two inside the
PaddingAutoencoder encoder and one in the classifier MLP head. We re-enable
these at inference time and average over `n_mc_samples` stochastic forward
passes (Gal & Ghahramani 2016).

There is one subtlety: `code/ERGO/ae_utils.predict` calls `model.eval()` at
the start of every call, which would silently disable our re-enabled
dropout. `evaluation/ergo_uncertainty.py` works around this by
monkey-patching `ae_utils.predict` for the duration of the MC sampling loop
with a variant that omits the `model.eval()` call. The original is restored
in a `finally` block, so the patch never leaks out of
`mc_dropout_predict_chunked`.

### Interpreting the std

- **Low mean, low std** → confident non-binder. Good.
- **High mean, low std** → confident binder. Good for targets, bad for decoys.
- **Mid mean, high std** → uncertain — model is on the fence.
- **High mean, high std** → flagged binder, but treat as soft positive.

The metrics step prints Pearson r between `ergo_mean` and `ergo_std`. A
strongly negative correlation (more confident at the extremes) is a
healthy calibration sign; a flat correlation (≈0) means the std is not
informative.
