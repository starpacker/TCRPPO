## 🎯 根本原因分析

### 1. 配置问题

```yaml
preserve_high_init_threshold: -0.5      # 阈值太低
preserve_high_init_tolerance: 0.05
preserve_high_init_weight: 3.0          # 惩罚权重太高
allow_stop_at_step0: true               # 允许立即STOP
```

**问题**:
- `-0.5` 还不算"真正好"的TCR（理想应该 >0.5）
- 3倍权重的退化惩罚过于严厉
- 允许step0 STOP给了模型"逃避"的机会

### 2. 采样策略问题

```yaml
online_tcr_pool_use_dynamic_bands: true
online_tcr_pool_elite_ratio: 0.35
online_tcr_pool_elite_min_affinity: -0.5
```

**问题**:
- 虽然启用了dynamic_bands，但实际采样没有遵循课程学习
- 39.5%的episodes是高InitA种子 (>-0.5)
- 过早挑战高难度编辑任务

### 3. Reward机制问题

对于InitA > -0.5的种子：
- 立即STOP → 获得正reward
- 尝试编辑但退化 → 遭受3倍惩罚

**结果**: 模型学到"风险规避"策略。

## 💡 解决方案：课程学习 (Curriculum Learning)

### 核心思想

**不要一开始就让模型编辑"接近完美"的TCR**

遵循"从易到难"的训练策略：

### 阶段1：早期 (Step 0-400K)
- **主要采样**: InitA < -1.5 的TCR
- **成功率**: 96.8%
- **平均改进**: +2.6
- **目标**: 让模型建立"我能改进TCR"的信心

### 阶段2：中期 (Step 400K-800K)
- **逐渐增加**: InitA ∈ [-1.5, -0.5] 的TCR
- **成功率**: 81.8%
- **平均改进**: +0.3
- **目标**: 继续强化编辑能力

### 阶段3：后期 (Step 800K+)
- **才挑战**: InitA > -0.5 的TCR
- **成功率**: 18.2%（但此时模型已有能力）
- **目标**: 学会在"已经很好"的基础上微调

### 方案C：分层的STOP策略

```yaml
# 新增：根据InitA分层允许STOP
stop_policy:
  - { init_affinity_min: 0.6,  allow_step0_stop: true  }  # 很好的可以立即STOP
  - { init_affinity_min: 0.0,  allow_step0_stop: false }  # 好的必须尝试
  - { init_affinity_min: -10,  allow_step0_stop: false }  # 差的必须尝试
```
